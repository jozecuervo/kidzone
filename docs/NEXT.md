# Kidzone Next Features

## Next Up

- Let the project shelf grow from `project.json` metadata with one card per
  mini-project.
- Make the first contributor path small: edit a starter project, preview it,
  and open a pull request.
- Add a second mini-game after the first starter proves the folder pattern.

## After That

- Decide whether mini-projects need a shared asset or helper library.
- Add lightweight checks for project metadata and broken relative links.
- Improve the landing page with kid-made project art as the shelf fills up.

## Guardrails

- Keep mini-projects GitHub Pages friendly.
- Prefer a project folder over a large framework jump.
- Do not add accounts, public sharing, or saved kid data until a project needs
  those decisions explicitly.
